Question:
Why do I get a NullPointerException in Java?

Model:
gpt-5.4

Answer:
A `NullPointerException` in Java happens when your code tries to use `null` as if it were a real object.

## What `null` means
`null` means “no object is assigned here.”

So if you do something like:

```java
String name = null;
System.out.println(name.length());
```

Java throws a `NullPointerException` because `name` does not point to a `String` object, and you’re trying to call `.length()` on it.

## Common causes
You usually get this when you:

- call a method on a `null` reference
- access a field on a `null` reference
- use array access on a `null` array
- unbox a `null` wrapper type like `Integer`

### Examples

#### 1. Calling a method on `null`
```java
Person p = null;
p.getName();   // NullPointerException
```

#### 2. Accessing a field on `null`
```java
Person p = null;
System.out.println(p.name);   // NullPointerException
```

#### 3. Null array
```java
int[] nums = null;
System.out.println(nums.length);   // NullPointerException
```

#### 4. Auto-unboxing
```java
Integer x = null;
int y = x;   // NullPointerException
```

## How to fix it
Make sure the object is initialized before using it.

```java
String name = "Alice";
System.out.println(name.length());
```

Or check for `null` first:

```java
if (name != null) {
    System.out.println(name.length());
}
```

## How to debug it
Look at the stack trace. It tells you:
- the line where the exception happened
- the chain of method calls that led there

Example:
```java
Exception in thread "main" java.lang.NullPointerException
    at Main.main(Main.java:5)
```

This means line 5 in `Main.java` tried to use a `null` reference.

## Modern Java note
Newer Java versions often give more helpful messages, like:

```java
Cannot invoke "String.length()" because "name" is null
```

That tells you exactly which variable was `null`.
